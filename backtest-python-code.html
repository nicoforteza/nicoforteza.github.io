Backtesting Algorithms with Python

  • by:Nicolás Forteza
  • date:23th September 2018

Let's see some code. Some quantitative finance professionals used C++ o Matlab to develop their programs that would backtest an strategy. Nowadays, Python seems to be the perfect language due to its simplicity and high-level approach. There are several libraries such as Numpy and Pandas that allow us to go very fast through matrix operations (well, Pandas needs to gain some performance...). The thing is, whether you are trying to test an algorithm you created, or to see if your model is doing well, you will have to go back in time and trigger the clauses that makes the model/strategy/program work as it is.

Although there are many platforms that do this work, you sometimes do not want to fall into any statistical biases such as in here. Therefore, coding for this purpose will give as a reward. In this case, the reward comes from the fact that we know all the pieces that are part of the process, so in case we want to improve something, its easy money. That's why I wanted to bring this sample code I made. In particular, I bring some code for practising factor investing. Several momentum strategies vs. a low volatility strategy. What the user needs to run the program is a matrix of returns. And thats it! The original post was published in Quantdare, a blog for quantitative finance. Cheers!
